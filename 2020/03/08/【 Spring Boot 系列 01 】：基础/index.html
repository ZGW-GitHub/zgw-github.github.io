一、常用注解

1、配置文件属性注入

  • 用在方法上：

@EnableConfigurationProperties

  • 用在类上：

2、配置相关

@Configuration

作用：

  • 指定当前类是一个配置类

@Import

作用：

  • 用于导入其他的配置类
  • 当我们使用 Import 的注解之后，有 Import 注解的类称为父配置类，而导入的称为子配置类 。

@PropertySource

作用：

  • 用于指定 properties 文件的位置

取值：

  • 关键字：classpath ，表示类路径下

3、依赖注入

@Autowired

作用：

  • 自动类型注入

使用位置：

  • 变量上 、方法上

@Qualifier

作用：

  • 按照 ID 注入

使用位置：

  • 变量上（ 需要与 Autowired 配合 ）、方法上

@Resource

作用：

  • 按照 ID 注入（ 可独立使用 ）

使用位置：

  • 变量上 、方法上

属性：

  • name
    用于指定 bean 的 id

@Value

作用：

  • 注入基本类型和String类型的数据

使用位置：

  • 变量上 、方法上

属性：

  • value
    用于指定数据的值。它可以使用 Spring 中 SpEL ( 也就是 Spring 的 EL 表达式 ）SpEL 的写法：#{表达式}​

4、IOC

@EnableAspectJAuroProxy

作用：

  • 启用 IOC 的注解配置

@Aspect

作用：

  • 标识当前类是一个切面类

@Before

作用：

  • 标识 前置通知

@After

作用：

  • 标识 后置通知

@AfterThrowing

作用：

  • 标识 异常通知

@AfterReturning

作用：

  • 标识 最终通知（ 方法最终结束后执行，相当于 finaly ）

@Around

作用：

  • 标识 环绕通知​

环绕通知的配置：

  • @Around("pt1()")
public Object aroundAdvice(ProceedingJoinPoint pjp){
    Object rtValue = null;
    try {
        // 1.获取参数
        Object[] args = pjp.getArgs();
        // 2.开启事务（前置通知）
        this.beginTransaction();
        // 3.执行方法
        rtValue = pjp.proceed(args);
        // 4.提交事务（后置通知）
        this.commit();
        // 返回结果
        return  rtValue;
    } catch (Throwable e){
        // 5.回滚事务（异常通知）
        this.rollback();
        throw new RuntimeException(e);
    } finally {
        // 6.释放资源（最终通知）
        this.release();
    }
}
<!--￼3-->

缺陷 ？：

  • 使用注解配置时，前置、后置、异常、最终，四大通知的执行顺序会絮乱（环绕通知不会），所以使用需慎重 。

    絮乱后的顺序：前置 –> 最终 –> 后置/异常

5、Bean 相关

改变作用域

@Scope
  • singleton
    单例的（默认值）
  • prototype
    原型
  • request
    作用于 web 应用的请求范围
  • session
    作用于 web 应用的会话范围
  • global-session
    作用于集群环境的会话范围（全局会话范围），当不是集群环境时，它就是 session

生命周期回调

@PreDestroy

作用：

  • 用于指定销毁方法
@PostConstruct

作用：

  • 用于指定初始化方法
