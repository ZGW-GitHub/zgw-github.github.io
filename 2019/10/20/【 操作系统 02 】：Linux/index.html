一、常用命令

1、浏览目录

pwd

显示所在目录

cd

  • cd

    进入用户登录时的工作目录

  • cd ~

    进入用户登录时的工作目录 ( 用户的根目录 )

  • cd -

    进入上次的目录

ls

ls [参数] [目录或文件]

  • -a

    显示所有文件 , 包括隐藏文件

  • -l

    显示文件详细信息

2、浏览文件

cat

滚屏查看文件内容

  • cat [参数] 文件名

将多个文件合并为一个文件

  • cat 文件名1 文件名2 > 文件名3 会覆盖文件3中原有内容

  • cat 文件名1 文件名2 >> 文件名3 将文件内容追加到文件3中

  • -b

    对输出内容的非空行标注行号

  • -n

    对输出内容的所有行标注行号

more

分屏显示文件内容 , 只能向下翻页

  • more [参数] 文件名

在管道中调用以实现各种命令输出内容的分屏显示

  • 如 :

    cat 文件名 | more 以分页方式查看文件的内容

  • -num

    num 是一个数字 , 指定每页显示的行数

  • +num

    指定从文件第 num 行开始显示

less

是 more 的改进版 , 可以上下翻页 , 甚至前后左右地移动

执行了 less 后就进入了 less 状态 , 按 q 可退出 less 命令

  • / xxx

    查找单词或字符

  • Enter

    向下移动一行

  • Space / d

    向下移动一页 / 移动半页

  • b / u

    向上移动一页 / 移动半页

  • 方向键

    前后左右移动

head

显示文件的开始部分 , 默认显示前10行

head [参数] 文件名

  • -n num

    显示文件的前 num 行

  • -c num

    显示文件的前 num 个字符

tail

显示文件的末尾部分 , 默认显示文件末尾 10 行内容

tail [参数] 文件名

  • -f

    不断刷新显示

  • -n num

    显示文件的末尾 num 行

  • -c num

    显示文件的末尾 num 个字符

  • +num

    从第 num 行开始显示文件内容

3、目录操作

mkdir

创建目录

mkdir [参数] 目录名

  • -p

    递归创建目录

  • -m

    配置目录权限

rmdir

删除空目录

rmdir [参数] 目录名

  • -p

    同时删除父目录 , 前提父目录中无其它目录或文件

    如 : rmdir -p dir2/dir1 删除 dir1 时 , 若 dir2 中无其它文件/目录 , 则将 dir2 一起删除

4、复制 目录 / 文件 cp

cp [参数] 源文件 目标文件

目标文件为 . 表示复制到当前目录

  • -a

    尽可能将文件状态 , 权限等属性按状态原状予以复制

  • -f

    若目标文件/目录存在 , 先删除再复制 , 删除不提醒用户

  • -i

    若目标文件/目录存在 , 提示是否覆盖已有文件/目录

  • -r

    递归复制目录

  • -R

    递归复制目录 , 复制过程中对特殊文件（管道文件，块设备文件，字符设备文件）会进行创建操作，而不是拷贝

  • -d

    若源文件为连接文件的属性(link file) , 则复制联结文件属性而非文件本身

  • -l

    不复制文件 , 只是进行硬式连接(hard link)的联结文件建立

  • -p

    连同文件属性一起复制过去

  • -s

    复制成为符号连接文件(symbolic link) , 亦即”快捷方式”文件

  • -u

    若目标文件比源文件旧才更新目标文件

5、操作文件

mv

文件/目录的移动和改名

mv [参数] 源文件/目录 目标文件/目录

  • -i

    提示是否覆盖目标文件/目录

  • -f

    不提示 , 直接覆盖目标文件/目录

如 :

mv test /usr/ 将当前目录下的 test 文件移动到 /usr/ 目录下 , 文件名不变

mv /usr/test /tt 将 /usr/test 文件移动到根目录下 , 并更名为 tt

rm

文件/目录的删除

rm [参数] 文件/目录名

  • -i

    提醒用户

  • -f

    不提醒用户

  • -r

    递归删除

touch

  1. 建立文件

  2. 更新文件的修改日期

touch [参数] 文件名/目录名

  • -d yyyymmdd

    将文件的存取和修改时间改为 yyyy 年 mm 月 dd 日

  • -a

    把文件的存取时间改为当前时间

  • -m

    把文件的修改时间改为当前时间

ln

建立两个文件之间的链接关系

  • 符号链接(软链接)

软链接类似于 Windows 的快捷方式

  • 硬链接

建立硬链接的两个文件 , 对其一内容的修改会影响到另一个文件

不能对目录建立硬链接

ln [参数] 源文件/目录 链接名

  • -s

    建立符号链接(软链接) , 不加该参数建立的为硬链接

6、解压缩

.gz 解压缩

压缩 : gzip [参数] 文件名

解压 : gunzip [参数] 文件名

  • -v

    显示压缩/解压比

tar

文件打包 , tar 将一系列文件归档到一个大文件中 , tar 是 Linux 常用的备份工具之一

tar [参数] 档案文件 文件列表

tar -zxvf xxx.tar.gz

  • -v

    显示压缩/解压比

  • -c

    压缩

  • -x

    解压

  • -f

    指定档案文件名

  • -d

    比较档案与当前目录中的文件

  • -r

    将文件追加到档案文件末尾

  • -z

    gzip 格式解压缩

  • -j

    bzip2 格式解压缩

zip

压缩

  • zip [参数] filename.zip 目录/*

    • -r

      递归压缩

解压

  • unzip [参数] 目录 filename.zip

    • -o

      不提示，直接覆盖

    • -d

      表示要解压到指明的目录

find [路径] [匹配表达式]

查找文件中包含指定字符串的行

grep [参数] 要查找的字符串 文件名

要查找的字符串 :

^ 表示行的开始

$ 表示行的结束

  • -i

8、VIM

三个模式：

  • 一般指令模式（Command mode）：VIM 的默认模式，可以用于移动游标查看内容；
  • 编辑模式（Insert mode）：按下 “i” 等按键之后进入，可以对文本进行编辑；
  • 指令列模式（Bottom-line mode）：按下 “:” 按键之后进入，用于保存退出等操作。

10、权限修改 chmod

chmod 选项 文件

数字表示法 :

  • 读取 ( ) 4
  • 写入 ( ) 2
  • 执行 ( ) 1

默认权限：

  • 文件默认权限：文件默认没有可执行权限，因此为 666，也就是 -rw-rw-rw- 。
  • 目录默认权限：目录必须要能够进入，也就是必须拥有可执行权限，因此为 777 ，也就是 drwxrwxrwx。

常用：

  • chmod -R 777 filename/dirname

    如：

    chmod -R 777 *

  • -F

2、系统监控 top

显示实时的进程列表 , 并实时监视系统资源

  • 第一行 :

    • 当前时间

    • 系统启动时间

    • 当前系统登录用户数目

    • 平均负载

      分别为： 1分钟、5分钟、15分钟内系统的平均负荷

  • 第二行 :

    • 启动的进程数
    • 运行的进程数
    • 挂起的(sleeping)进程数
    • 终止的(stopped)进程数
    • 无用的(zombie)进程数

  • 第三行 : CPU 的使用情况

    • 系统占用比例
    • 用户使用比例
    • 闲置比例 ( idle )

  • 第四行 : 物理内存使用情况

    • 总的可用内存
    • 已用
    • 空闲
    • 缓冲区占用内存

  • 第五行 : 交换分区使用情况

    • 总的交换分区
    • 已用
    • 空闲
    • 高速缓存大小

  • 第六行 :

    • PID 进程标识号
    • USER 进程所有者的用户名
    • PR 进程的优先级别
    • NI 进程的优先级别数值
    • VIRT 进程占用的虚拟内存值
    • RES 进程占用的物理内存值
    • SHR 进程时使用的共享内存值
    • S 进程的状态
      • S 休眠 sleep
      • R 运行 run
      • Z 僵死状态 zombin
      • N 进程优先值是负数
    • %CPU
    • %MEM 进程占用物理内存百分比
    • TIME+ 进程启动后占用的总 CPU 时间
    • COMMAND 进程启动的启动命令名称

top 使用过程中 , 可以使用一些交互命令 :

  • 空格

    立即刷新

  • P

    根据 CPU 使用大小进行排序

  • q

    退出 top 命令

  • . . .

选项与参数

top [-d 数字] | top [-bnp]

  • -d

    接秒数 , 刷新时间

  • -b

    以批量的方式运行 top , 通常搭配数据流重导向来将批量的结果输出为文件

  • -p

    指定某些 PID 来进行观察监测

  • -n

    与 -b 搭配 , 意义 : 需要几次 top 的输出结果

1、进程查看 ps / pstree

ps [参数]

  • ps -aux 查看系统所有的进程数据
  • ps -lA 查看系统所有的进程数据
  • ps -axjf 连同部分进程树状态一起查看
  • ps -l 查看自己的进程
  • ps aux | grep xxx 查看特定的进程

参数 :

  • -A

    所有的进程均显示 , 与 -e 有相同作用

  • -e

    所有的进程均显示

  • -a

    与 terminal 无关的所有进程

  • -u

    与有效使用者相关的进程

  • -x

    通常与 -a 一起使用 , 可列出较完整信息

输出格式规划 :

  • l

    较长 , 较详细地将该 PID 的信息列出

  • j

    工作的格式

  • -f

    做一个更为完整的输出

2、进程管理 kill

kill -代号 PID

代号 名称 作用 :

  • 1 sighup

    启动被终止的进程 , 可让该 PID 重新读取自己的配置文件 , 类似重新启动

  • 2 sigint

    相当于 Ctrl + C 中断一个程序的进行

  • 9 sigkill

    强制中断一个进程的执行 , 可能会由半成品的产生

  • 15 sigterm

    正常结束进程 , 若进程出错无法正常结束 , 则使用该命令也无用

  • 17 sigstop

    相当于 Ctrl + Z 暂停一个进程的执行

八、防火墙

CentOS 6.x 版本使用的是 iptables 防火墙。

CentOS 7 使用的是 firewalld 防火墙，如果想换回 iptables 防火墙，可关闭 firewalld 并安装 iptables 。

iptables

  • 查看防火墙状态：

    service iptables status

  • 开启防火墙：

    service iptables start

  • 关闭防火墙：

    service iptables stop

firewalld

  • 停止 firewall

    systemctl stop firewalld.service

  • 禁止 firewall 开机启动

    systemctl disable firewalld.service

  • 重启防火墙

    firewall-cmd --reload

  • 查看默认防火墙状态（ 关闭后显示 notrunning，开启后显示 running ）

    firewall-cmd --state

  • 查看开放的端口

    firewall-cmd --list-all

  • 设置开放的端口

    sudo firewall-cmd --add-port=80/tcp -permanent

    firewall-cmd --add-service=http -permanent
