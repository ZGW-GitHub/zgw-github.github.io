模糊查询：

方式一：

  • 传实参时传入 % 等模糊字符

    生成的 SQL 语句为 ? 占位符

方式二：

  • 映射配置文件的 SQL 语句添加 %

    生成的 SQL 语句为 %xxx

  • 注意：

    select * from user where username like ‘%{value}%’

    %{value}% 必须为 value

#{} 与 ${} ：

相同：

  • 都可以接收简单类型值或 pojo 属性值

不同：

  • #{}

    • 占位符设置值，可防止 SQL 注入
    • parameterType 传输单个简单类型值时，{} 内可写任意字符串

  • ${}

    • 拼接字符串
    • parameterType 传输单个简单类型值时，{} 内只能写 value

  2. 通过 resultMap 标签配置结果集类名与数据库列名的对应关系

六、多表查询

一对一（多对一）

如：一个 User 有一个 Key（账号）

方式一：

定义一个 UserKey 类继承 User 类，并将 Key 类的所有字段添加进 UserKey 类中，来实现封装用户与该用户的账号信息

八、缓存

Mybatis 提供了缓存策略，通过缓存策略来减少数据库的查询次数，从而提高性能。

Mybatis 中缓存分为一级缓存，二级缓存。

一级缓存

  • 概念：
    • 它指的是 Mybatis 中 SqlSession 对象的缓存。当我们执行查询之后，查询的结果会同时存入到 SqlSession 为我们提供一块区域中。
    • 该区域的结构是一个 Map 。当我们再次查询同样的数据，mybatis 会先去 sqlsession 中查询是否有，有的话直接拿出来用。
  • 缓存的清空：
    • 当 SqlSession 对象消失时，mybatis 的一级缓存也就消失了。
    • SqlSession 对象调用 clearCache() 方法时，也可以清空 SqlSession 对象中缓存。
    • 当调用 SqlSession 的 修改、添加、删除、commit()、close() 等方法时，会清空一级缓存。

二级缓存

  • 概念：

    • 它指的是 Mybatis 中 SqlSessionFactory 对象的缓存。
    • 由同一个 SqlSessionFactory 对象创建的 SqlSession 共享其缓存。

  • 注意：

    • 二级缓存中存放的是数据不是对象，其每次查询时会创建一个新的对象并将缓存中的数据封装进去，所以每次查询即使要查询的值未改变，查询所获取的对象也不相同

  • 二级缓存的配置使用：

    • 第一步：让 Mybatis 框架支持二级缓存（在主配置文件中配置延迟加载的全局开关）

    • 第二步：让当前的映射文件支持二级缓存（在映射配置文件中配置）

    • 第三步：让当前的操作支持二级缓存（在映射配置文件中的具体操作标签(如：select)中配置 userCache 属性值为 “true”）

1、Results

解决结果集类名与数据库列名不一致

2、One / More

实现多表查询

注解的属性：

  • select

  • fetchType

      表示此语句延迟加载

    • FetchType.EAGER

      表示此语句立即加载

    • FetchType.DEFAULT

      表示在上面两个任选其一

一级缓存

一级缓存默认开启支持，不用配置

二级缓存

  • 第一步：

    让 Mybatis 框架支持二级缓存（在主配置文件中配置延迟加载的全局开关）

  • 第二步：

    在 dao 类上使用 @CacheNamespace(blocking=true) 注解即可
