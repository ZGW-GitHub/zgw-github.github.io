③、UsernamePasswordAuthenticationFilter

经过 AbstractAuthenticationProcessingFilter 的处理后，请求来到了 UsernamePasswordAuthenticationFilter ：

首先，UsernamePasswordAuthenticationFilter 的构造方法，初始化了一个 AntPathRequestMatcher 类，当 AbstractAuthenticationProcessingFilter 调用 requiresAuthentication 方法筛选 Filter 时，会通过该匹配器来匹配请求。

当匹配成功，UsernamePasswordAuthenticationFilter 会执行 attemptAuthentication 方法执行认证：

attemptAuthentication 方法会先判断请求是否为 POST 请求，然后会从请求中获取提交的用户名和密码。并用获取到用户名和密码，构建一个未认证的 AuthenticationToken 并将发出该请求的 IP 及 Session ID 存入 AuthenticationToken。

然后，请求会进入 AuthenticationManager 挑选能处理该请求的 AuthenticationProvider 进行认证。

④、AuthenticationManager

当 AuthenticationManager 通过循环遍历出能处理该 Authentication 的 AuthenticationProvider 后，会调用其 authenticate 方法，进行真正的认证操作。

其中处理表单认证的 AuthenticationProvider 为 AbstractUserDetailsAuthenticationProvider 。
