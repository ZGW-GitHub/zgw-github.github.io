【 必备技术 02 】：正则表达式
发表于 2019-09-02| 更新于 2020-03-21|必备技术
阅读量:

一、常用

  • ^

    一行的开始

  • $

    结束

  • 字符组：

    • [ae]

      匹配：a / e

    • 字符组元字符

      • -

        • 注意：

          只有在字符组内部，它才是字符组元字符；

          当其处于字符组的开头，它就只是一个普通字符

        • 使用实例：

          表示一个范围

          [0-9a-fA-F_?!]

      • ^

        • 注意：

          只有其在字符组内部，且紧跟 [ 之后，才有作用；

          「 [ ^d ] 」：意思并不是：“只有当这个位置不是 d 时才匹配”，而是说：“匹配一个不等于 d 的字符”。其中的差别很微妙，但很重要。例如：前面的概念可以匹配一个空行，而「 [ ^d ] 」则不行。

        • 使用实例：

          [^0-9]

          匹配 0-9 以外的任何字符

  • 「ab[cd]」

    匹配：abc / abd

  • .

    是用来匹配任意字符的字符组的简便写法

  • |

    或

    • 示例：「ab[cd]」、「abc|abd」、「ab(c|d)」

  • ?

    可选项元素

    • 示例：
      • 「colou?r」匹配：color / colour
      • 「colo(uu)?r」匹配：color / colouur

  • +

    表示：之前紧邻的元素出现一次或多次，之前紧邻的元素可以是字符组

  • *

    表示：之前紧邻的元素出现任意多次，或者不出现，之前紧邻的元素可以是字符组

  • \

二、Perl

  • \t

    匹配制表符

  • \n

    匹配换行符

  • \f

    ASCII 的进纸符

  • \b

    有时：退格符（在字符组中）

    有时：单词分界符

  • \s

    能表示所有表示“空白字符”的字符组，其中包括：空格符、制表符、换行符、回车符

  • (?: · · · )

    与 ( · · · ) 对比： ( · · · ) 用于分组和捕获，而 (?: · · · ) 只分组不捕获

  • 修饰符：

    • i

      Perl 实现忽略大小写 ：m/ · · · /

    • g

      全局匹配

    • x

      宽松排列的表达式

三、兼容问题

  • < 与 \>

    「<cat\>」：匹配单词 cat

  • {2,5}

    规定重现次数的范围：区间

四、其它

-i 参数：表示忽略大小写
