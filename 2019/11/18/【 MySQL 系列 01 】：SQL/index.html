一、SQL

Structured Query Language ：结构化查询语言

其实就是定义了操作所有关系型数据库的规则。

SQL 的作用：

  • 是一种所有关系型数据库的查询规范，不同的关系型数据库都支持。
  • 通用的数据库操作语言，可以用在不同的关系型数据库中。
  • 不同的数据库 SQL 语句有一些区别。

SQL 通用语法：

  • SQL 语句可以单行或多行书写，以分号结尾。

  • 可使用空格和缩进来增强语句的可读性。

  • MySQL 数据库的 SQL 语句不区分大小写，关键字建议使用大写。

  • 3 种注释：

  • 登录

SQL 语句分类：

  • DDL ( Data Definition Language ) 数据定义语言

    增删改查 数据库 / 表

  • DML ( Data Manipulation Language ) 数据操作语言

    增删改表中数据，及创建表

  • DQL ( Data Query Language ) 数据查询语言

    查询表中数据

  • DCL ( Data Control Language ) 数据控制语言

    用来定义数据库的访问权限和安全级别，及创建用户

SQL 数据类型：

整数

  • tinyint

    微整型：很小的整数 ( 1字节 )

  • smallint

    小整型：小的整数 ( 2字节 )

  • mediumint

    中整型：中等长度的整数 ( 3字节 )

  • int

    整型：整数类型 ( 4字节 )

小数

  • float

    单精度浮点数，占 4 个字节

  • double

    双精度浮点数，占 8 个字节

字符串

  • char

    固定长度的字符串，无论使用几个字符都占满全部，M 为 0~255 之间的整数

  • varchar

    可变长度的字符串，使用几个字符就占用几个，M 为 0~65535 之间的整数

日期

  • time

    表示时间类型

  • date

    表示日期类型。只包含年月日，yyyy-MM-dd

  • datetime

    表示日期和时间类型。包含年月日时分秒，yyyy-MM-dd HH:mm:ss

  • timestamp

    时间戳类型。包含年月日时分秒，yyyy-MM-dd HH:mm:ss此类型数据当将来不给这个字段赋值，或赋值为null，则默认使用当前的系统时间，来自动赋值​

大二进制

  • tinyblob Big Large Object

    允许长度 0~255 字节

  • blob

    允许长度 0~65535 字节

  • mediumblob

    允许长度 0~167772150 字节

  • longblob

    允许长度 0~4294967295 字节

大文本

  • tinytext

    允许长度 0~255 字节

  • text

    允许长度 0~65535 字节

  • mediumtext

    允许长度 0~167772150 字节

  • longtext

    允许长度 0~4294967295 字节
